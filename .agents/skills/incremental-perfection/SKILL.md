# Skill: Incremental Perfection
# Trigger: On code generation or modification tasks

## Directives
1. Break down every complex feature or refactoring task into the smallest possible logical units (micro-steps).
2. Execute and implement ONLY ONE micro-step or single component/function at a time.
3. DO NOT write or modify code for multiple files or sub-modules simultaneously.
4. After completing a micro-step, verify its local execution and prompt the user to make a clean Git commit before proceeding to the next step.
